push到github远程仓库时，每次都要输入用户名和密码的问题

发表于 2018-02-26 | 分类于 git

过完年回来，发现push 到 github远程仓库时，每次都需要输入用户名和密码，这也太麻烦了，于是查了下解决方法。

原因及解决

原因是使用了 https 方式 push

使用 git remote -v 查看对应的仓库地址, 如果返回的结果是以下这种格式的, 则是https方式
